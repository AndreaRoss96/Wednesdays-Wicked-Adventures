# Social Contract

**Team:** Group 3 - Wednesday's Wicked Adventures
**Project:** Horror Park Booking System
**Module:** DevOps Project Management

---

## Team Values

- **Respect** - We treat all team members with respect and value diverse perspectives
- **Transparency** - We communicate openly about progress, blockers, and challenges
- **Accountability** - We take ownership of our assigned tasks and deliver on commitments
- **Collaboration** - We support each other and share knowledge freely

---

## Communication

### Channels
- **Primary:** Microsoft Teams / Discord (as agreed by team)
- **Code Reviews:** GitHub Pull Requests
- **Task Tracking:** Jira

### Response Times
- Messages during working hours: respond within 24 hours
- Urgent issues: notify via agreed channel immediately

### Meetings
- Attend all scheduled stand-ups and sprint ceremonies
- Notify the team in advance if unable to attend
- Come prepared with updates on assigned tasks

---

## Working Agreements

### Git Workflow
- Create feature branches using naming convention: `feature/SCRUM-XX-description`
- Never commit directly to `main` or `UAT` branches
- All changes require a Pull Request with at least 1 approval
- CI checks must pass before merging

### Code Quality
- Write clean, readable code with meaningful names
- Test your changes locally before creating a PR
- Address review comments promptly

### Task Management
- Update Jira tickets when starting, completing, or blocking on tasks
- Break down large tasks into smaller, manageable pieces
- Raise blockers early - don't wait until the last moment

---

## Conflict Resolution

1. **Direct Discussion** - Address issues directly with the person involved
2. **Team Discussion** - If unresolved, bring to the team during a meeting
3. **Escalation** - If still unresolved, involve the module lecturer

---

## Definition of Done

A task is considered "Done" when:
- [ ] Code is complete and tested locally
- [ ] Pull Request is created and approved
- [ ] CI pipeline passes all checks
- [ ] Documentation is updated (if applicable)
- [ ] Jira ticket is moved to Done
